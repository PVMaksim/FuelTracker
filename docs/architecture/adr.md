# ADR — Принятые архитектурные решения

## ADR-001: PWA вместо нативного iOS приложения

**Статус:** Принято  
**Дата:** 30.04.2026

**Контекст:** Нужно мобильное приложение для iPhone без сложного стека.

**Решение:** Progressive Web App (Next.js)

**Обоснование:**

- Не нужен App Store и Apple Developer ($99/год)
- Доступ к камере через WebAPI (для OCR)
- Один кодовой базой покрываются веб и мобайл
- Знакомый TypeScript/React стек
- Установка через Safari → «На экран Домой»

**Отклонённые альтернативы:**

| Альтернатива | Причина отказа |
|---|---|
| Swift/SwiftUI | Требует Mac + Xcode + $99/год, сложный стек |
| React Native | Overkill для одного разработчика, нужен Expo |
| Telegram Bot | Нет нативного доступа к камере, текстовый UI |
| Telegram Mini App | Зависимость от Telegram, сложнее офлайн |

## ADR-002: Claude API для OCR вместо Google Vision / Tesseract

**Статус:** Принято  
**Дата:** 30.04.2026

**Контекст:** Нужно распознавать чеки АЗС на русском языке.

**Решение:** Anthropic Claude API (vision)

**Обоснование:**

- Высокая точность на русских чеках (понимает контекст, не просто OCR)
- Не нужен отдельный аккаунт — тот же провайдер что и для разработки
- Возвращает структурированный JSON напрямую (не нужен пост-парсинг)
- ~$0.01/фото — дёшево для личного использования

**Отклонённые альтернативы:**

| Альтернатива | Причина отказа |
|---|---|
| Tesseract (self-hosted) | Средняя точность для рукописных/нестандартных чеков |
| Google Vision API | Отдельный аккаунт GCP, меньше контекста |
| Mindee | Free tier 250/мес — мало для долгосрочного использования |

## ADR-003: Статический API-ключ вместо OAuth

**Статус:** Принято  
**Дата:** 30.04.2026

**Контекст:** Аутентификация между PWA фронтендом и FastAPI бэкендом.

**Решение:** Один статический ключ в заголовке `X-API-Key`

**Обоснование:** Один пользователь (разработчик). OAuth — излишняя сложность для личного инструмента.

**Риски и митигация:**

- Ключ в `NEXT_PUBLIC_*` виден в исходниках клиента → **митигация:** Nginx ограничивает доступ к `/api/` только с нужных Origins; данные не секретны
- При компрометации ключ легко сменить в `.env` → редеплой

## ADR-004: Офлайн-first через IndexedDB (Dexie.js)

**Статус:** Принято

**Контекст:** Приложение должно работать без интернета (АЗС в зоне плохого сигнала).

**Решение:** IndexedDB через Dexie.js, синхронизация при восстановлении сети.

**Обоснование:**

- Dexie.js — лучший API поверх IndexedDB (промисы, запросы, миграции схемы)
- Стандартный Web API — работает в любом браузере без зависимостей
- `local_id` + дедупликация на сервере решают проблему повторной синхронизации
