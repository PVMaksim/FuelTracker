# Импорт данных из Apple Numbers

Скрипт `scripts/import_numbers.py` позволяет импортировать исторические данные заправок из таблицы Numbers.

## Шаг 1 — Экспортировать CSV из Numbers

1. Открыть файл `Расходы_питер.numbers`
2. **Файл → Экспортировать в → CSV...**
3. Сохранить файл, например `~/Downloads/expenses.csv`

## Шаг 2 — Dry run (без реального импорта)

```bash
python scripts/import_numbers.py \
  --file ~/Downloads/expenses.csv \
  --api-url http://localhost:8000/api/v1 \
  --api-key localdev-key \
  --dry-run
```

Скрипт покажет первые строки файла и попросит указать номера колонок:

```
📋 Превью файла:
#     [0] Дата           [1] Описание       [2] Сумма
      30.04.2026         Заправка Лукойл    2340
      29.04.2026         Еда                450

Колонка с пробегом (км) (введи номер [0-5]): 
```

## Шаг 3 — Реальный импорт

```bash
python scripts/import_numbers.py \
  --file ~/Downloads/expenses.csv \
  --api-url https://your-domain.com/api/v1 \
  --api-key YOUR_API_KEY \
  --odometer-col 3 \
  --cost-col 2 \
  --price-col 4 \
  --date-col 0
```

## Параметры скрипта

| Параметр | Обязательный | Описание |
|----------|-------------|----------|
| `--file` | ✅ | Путь к CSV файлу |
| `--api-url` | ✅ | URL API |
| `--api-key` | ✅ | API ключ |
| `--dry-run` | — | Тестовый запуск без импорта |
| `--odometer-col` | — | Номер колонки с пробегом (0-based) |
| `--cost-col` | — | Номер колонки с суммой |
| `--price-col` | — | Номер колонки с ценой за литр |
| `--date-col` | — | Номер колонки с датой |

!!! info "Интерактивный режим"
    Если не передать `--*-col` параметры — скрипт в интерактивном режиме
    покажет превью и попросит выбрать колонки.

## Поддерживаемые форматы дат

- `30.04.2026`
- `30.04.26`
- `2026-04-30`
- `30/04/2026`

## Что делать если Numbers не хранит цену за литр

В таблице `Расходы_питер.numbers` цена за литр может отсутствовать.  
В таком случае:

1. Передай `--price-col` несуществующей колонкой — скрипт пропустит строки без цены
2. Или предварительно добавь колонку цены в Numbers перед экспортом
3. Или после импорта отредактируй записи вручную через API

!!! tip "Первый пробег"
    Скрипт передаёт `local_id = import-{line_number}`, поэтому повторный запуск
    не создаст дублей — сервер пропустит уже существующие записи.
